## From FEM to Neural Operators: Drawing a parallel between learning a Differential Equation using Finite Element Method, PINNs, and Neural Operators


### Overview

This repository accompanies the Spring School course:
[From FEM to Neural Operators: Scientific Machine Learning for Computational Mechanics](https://www.uni-weimar.de/springschool/de/16/from-fem-to-neural-operators-scientific-machine-learning-for-computational-mechanics/). Hosted at Bauhaus-Universität Weimar, this course explores a transformative shift in computational mechanics — from traditional Finite Element Methods (FEM) to Neural Operators and Scientific Machine Learning (SciML). Traditionally, FEM has been the backbone of solving partial differential equations (PDEs). Today, emerging AI-based approaches such as Physics-Informed Neural Networks (PINNs) and Neural Operators are redefining how we model, simulate, and accelerate physical systems. This repository is designed as a hands-on companion, guiding you through this transition. The recorded videos are on the [here](https://livejohnshopkins-my.sharepoint.com/:f:/g/personal/sgoswam4_jh_edu/IgDwNCByLHC6RYyYEWxxiWzYAV4sQcEvV_osZKDnH6Zzgek?e=p01h76)

### What You’ll Learn

By working through this repository, you will:
- 🧠 Understand the foundations of FEM and Scientific Machine Learning
- 🔁 Learn mappings from discrete solvers → operator learning frameworks
- ⚡ Train Neural Operators for PDE surrogates
- ⚖️ Benchmark FEM vs ML-based solvers (accuracy, speed, generalization)
- 🧩 Build intuition for when to use what

The philosophy is simple: Use FEM to understand physics - use Neural Operators to scale it.

### Why This Matters
Scientific Machine Learning is not replacing FEM — it is augmenting it.
This repository reflects a key paradigm:
- FEM → robust, physics-grounded, but computationally expensive
- PINNs and Neural Operators → fast, generalizable, data-driven

The future lies in hybrid solvers that combine both.
